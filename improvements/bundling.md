# Bundling for Serverless Distribution

## Problem

The project uses ES modules (`import`/`export`). Browsers block module imports when opening `index.html` via `file://` due to CORS restrictions. A local server is required during development, but for distribution the app should work by just double-clicking the HTML file.

## Solution: esbuild

Use esbuild to bundle all source files into a single `bundle.js` with no `import`/`export` statements. The bundled output runs as a classic `<script>` tag and works from `file://`.

- Source files stay as ES modules (unchanged)
- Development still uses the live server at `localhost:5500`
- Run one command before distributing to produce the bundle

## Prerequisites

- Node.js (already installed)

## Setup Steps

### 1. Initialize package.json

```
npm init -y
```

Creates `package.json` — a manifest that records which tools the project uses.

### 2. Install esbuild

```
npm install --save-dev esbuild
```

Downloads esbuild into `node_modules/` and records it in `package.json` under `devDependencies`.

### 3. Add build script to package.json

Edit `package.json` and add to the `"scripts"` section:

```json
{
  "scripts": {
    "build": "esbuild main.js --bundle --outfile=dist/bundle.js --format=iife"
  }
}
```

### 4. Create dist/index.html

A copy of `index.html` with the script tag changed:

```html
<!-- Instead of: -->
<script type="module" src="main.js"></script>

<!-- Use: -->
<script src="bundle.js"></script>
```

### 5. Copy static assets into dist/

The build should also copy `main.css` and the `assets/` folder into `dist/`.

## Usage

```
npm run build
```

This produces:

```
dist/
├── index.html     (classic script tag, no modules)
├── bundle.js      (all JS concatenated, self-contained)
├── main.css
└── assets/
    ├── fonts/
    └── svgData/
```

Zip `dist/` and distribute. Recipients double-click `index.html` — no Node.js, no server needed.

## Git

- `node_modules/` → add to `.gitignore` (regenerated by `npm install`)
- `dist/` → optional to commit; can always be regenerated with `npm run build`

## Minification (optional)

Add `--minify` to strip whitespace, shorten variable names, and remove comments. This makes the bundle smaller and harder to read:

```json
{
  "scripts": {
    "build": "esbuild main.js --bundle --outfile=dist/bundle.js --format=iife --minify"
  }
}
```

- No performance cost — the output is functionally identical
- Deters casual inspection of the source code (beautifiers exist, but the result is still hard to follow without original names)
- Does not generate source maps, so the original structure stays private

## Notes

- esbuild is extremely fast (milliseconds)
- `--format=iife` wraps everything in a self-executing function to avoid global scope pollution
- Source files are never modified — bundling is a separate output step
- If the project adds dynamic imports or circular dependencies in the future, esbuild handles those automatically
- **If this bundling setup is implemented, update `.kiro/steering/architecture.md`** — it currently states "no bundler" and "no external libraries or dependencies". The architecture doc should reflect that esbuild is used as a dev dependency for distribution builds (even though source files remain unbundled ES modules during development).
